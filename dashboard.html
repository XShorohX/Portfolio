Дашборд строительной готовности

Разработка системы цифрового мониторинга строительных объектов

Проект

Компания участвовала в проекте по цифровизации строительной отрасли Москвы. Существующая система мониторинга строительства базировалась на использовании эксель таблиц, что создавало значительные неудобства для всех участников процесса. Отсутствовала автоматизация отчётности и эффективный контроль сроков строительства.

На момент моего подключения к проекту уже были спроектированы некоторые основные экраны, которые нуждались в доработке.

Моя роль

Ведущий UX/UI дизайнер

Задачи

  • Разработка цифрового решения для мониторинга строительных объектов
  • Трансформация устаревшей системы отчетности на основе таблиц в современный цифровой продукт
  • Создание удобного инструмента для контроля сроков строительства и формирования отчетности

Цели и критерии успеха

  • Снизить случаи срыва сроков по объектам
  • Ускорить принятие решений руководством

Исследование

  • Изучила таблицы, которые использовались до цифровизации (Excel/Word)
  • Провела интервью с представителями ключевых ролей: проектными менеджерами, представителями заказчика и пользователями
Выявлено
  • Отчеты формируются вручную, без единых шаблонов
  • Нет целостного представления о статусе объектов
  • Визуально невозможно быстро определить "проблемные" объекты

Прототипирование

Совместно с аналитиком и менеджером построила пользовательские сценарии по поиску объекта со срывом срока, формированию отчета по объектам.

Собрала кликабельные прототипы, чтобы провести юзабилити-тесты и выбрать наилучшие сценарии поиска объекта, сортировки, перехода в карточку. Собирала обратную связь и дорабатывала интерфейс.

Результаты

На главной странице сервиса отображается дашборд, чтобы отследить статистику по объектам

В каждый блок дашборда можно провалиться и увидеть список объектов

Улучшена структура фильтров

Было:

Фильтр в реестре объектов занимал много места сверху+не был полным

Стало:

В результате интервью с проектными менеджерами определила, что основная часть пользователей находит нужные объекты через поиск по ключевой информации (адрес, наименование и т.д.). Остальные фильтры используются редко, но остаются важными для отдельных сценариев

Поэтому я перенесла фильтры в одну кнопку (“расширенные фильтры”), чтобы:

  • снизить визуальную перегрузку интерфейса
  • сфокусировать внимание на поиске
  • сохранить доступ к фильтрации для тех, кому она действительно нужна
Переработан формат реестра объектов

Был добавлен второй вариант отображения списка объектов, где видны все этапы и их текущее состояние

Добавлены визуальные индикаторы

Например, было важно отображать кем принято решение о переносе срока, поэтому, чтобы не перегружать карточку объекта я убрала информацию в попа

Итоги в цифрах

  • Срывы сроков по объектам снизились на 42%
  • Время принятия решений руководством сократилось в 2 раза
  • Сервис интегрирован в процесс стройконтроля как единый источник статуса объектов
